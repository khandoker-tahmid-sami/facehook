****************Rules to save access token, refreshtoken, and auth details***************************

The Options
1. localStorage — Most Common but Risky

localStorage.setItem("accessToken", token);
localStorage.getItem("accessToken");
Persists even after browser closes
Vulnerable to XSS attacks — if any malicious script runs on your page, it can steal the token

2. sessionStorage — Slightly Safer

sessionStorage.setItem("accessToken", token);
Cleared when the tab closes
Still vulnerable to XSS — same problem as localStorage

3. HttpOnly Cookie — Most Secure

Set-Cookie: accessToken=xyz; HttpOnly; Secure; SameSite=Strict
Set by the server, not JavaScript
JavaScript cannot read it at all — XSS attacks can't steal it
This is the recommended approach for production apps

4. Memory (React state / Zustand / Redux)

const [accessToken, setAccessToken] = useState(null);
Safest from XSS — never touches the browser storage
Lost on page refresh — user has to login again
The Best Practice (Industry Standard)
Token	Where to store
Access Token	Memory (React state / Zustand)
Refresh Token	HttpOnly Cookie (set by server)
Why this combination?

Access token lives in memory — XSS can't steal it, but it's lost on refresh (that's okay)
Refresh token is in HttpOnly cookie — only the server can read it, used to get a new access token silently when page reloads
For Your Project (Practical Advice)
Since you are learning/building, using localStorage is fine for now — but know it's not production-safe. When you are ready to make it production-ready, switch to the Memory + HttpOnly Cookie approach.


// For now — simple and works for learning
localStorage.setItem("accessToken", token);

// Later — proper way
// Server sets HttpOnly cookie for refresh token
// You store access token in Zustand/Redux store
Simple Rule to Remember
Never store sensitive tokens where JavaScript can read them in production. HttpOnly cookies are invisible to JavaScript — that's exactly what makes them safe.

*****************************Reason to use useDebugValue in useAuth file***********************************

What is useDebugValue?
It is a React hook only for developer tools. It has zero effect on your actual app — users never see it.

It adds a readable label to your custom hook inside React DevTools browser extension.

Without useDebugValue
In React DevTools you see:


useAuth
  (no useful info)
With useDebugValue

useAuth: "User logged in"   ← you can see auth state instantly
The second argument is a performance optimization

useDebugValue(auth, (auth) => auth?.user ? "User logged in" : "User logged out")
//                  ↑
//            this function only runs when DevTools is open
If you wrote it like this instead:


useDebugValue(auth?.user ? "User logged in" : "User logged out") // ❌ runs always
That calculation runs every render, even when DevTools is closed. The second argument (formatter function) makes it lazy — only runs when a developer actually has DevTools open.

Simple Rule
useDebugValue
Who sees it	Only developers with React DevTools
Affects app?	Never
When to use	In custom hooks to show useful state at a glance
Second argument	A formatter function — runs lazily only when DevTools is open
Think of it as a sticky note for developers — it helps you debug faster but means nothing to the real app.

2. We need to extend axios to make the user logged in thoughout the website. To update the auth token using calling
the refresh token api we need to extend axios. Axios provides interceptor feature. When can intercept in request and response time. 



Server will always check - does request header have auth token or not. 

This is the extension of axios with interceptor:

import axios from "axios";
import { api } from "../api";
import { useEffect } from "react";
import { useAuth } from "./useAuth";


export const useAxios = () =>{

    const {auth, setAuth} = useAuth()
    useEffect(() =>{

        //add a request incepter
       const requestIntercept =  api.interceptors.request.use(
            (config) =>{
                const authToken = auth?.authToken;

                if(authToken) {
                    config.headers.Authorization = `Bearer ${authToken}`
                }

                return config;
            },
            (error) => Promise.reject(error)
        )


        //add a response interceptop
        const responseIntercept = api.interceptors.response.use(
            (response) => response,
            async (error) => {
                const originalRequest = error.config;

                console.log(`OriginalRequest: ${originalRequest}`)
                if(error.response?.status === 401 && !originalRequest._retry){
                    originalRequest._retry = true

                    const refreshToken = auth?.refreshToken
                    const response = await axios.post(`${import.meta.env.VITE_SERVER_BASE_URL}/auth/refresh-token`, {refreshToken})
                        .catch(() => {
                            setAuth(null)
                            return Promise.reject(error)
                        })

                    const {token, refreshToken: newRefreshToken} = response.data

                    setAuth({...auth, authToken: token, refreshToken: newRefreshToken})
                    originalRequest.headers.Authorization = `Bearer ${token}`

                    return api(originalRequest)
                }
            }
        )

        return () =>{
            api.interceptors.request.eject(requestIntercept);
            api.interceptors.response.eject(responseIntercept);
        }

    },[auth, setAuth])

    return {api}
}

A lazy initializer function in React is a performance optimization technique used within the useState (and useReducer) hook to defer expensive, one-time computations until the component's initial render. Instead of passing a direct value, you pass a function that returns the initial value. React executes this function only once, when the component mounts, and ignores it during subsequent re-renders.

******************** File upload technique ****************************
as html input type file upload give a button. But if we want to use our custom button to open a image we need to use useRef, to grap that input element and open image using our custom button.



import { useRef, useState } from "react";
import { actions } from "../../actions";
import EditIcon from "../../assets/icons/edit.svg";
import { useAxios } from "../../hooks/useAxios";
import { useProfile } from "../../hooks/useProfile";

export const ProfileImage = () => {
  const { state, dispatch } = useProfile();
  const { api } = useAxios();
  const [bio, setBio] = useState(state?.user?.bio);
  const [editBio, setEditBio] = useState(false);

  const fileUploadRef = useRef();

  const handleImageUpload = (event) => {
    event.preventDefault();

    fileUploadRef.current.addEventListener("change", updateImageDisplay);
    fileUploadRef.current.click();
  };

  const updateImageDisplay = async () => {
    const formData = new FormData();
    for (const file of fileUploadRef.current.files) {
      formData.append("avatar", file);
    }

    try {
      const response = await api.post(
        `${import.meta.env.VITE_SERVER_BASE_URL}/profile/${state?.user?.id}/avatar`,
        formData,
      );

      if (response.status === 200) {
        dispatch({ type: actions.profile.IMAGE_UPDATED, data: response.data });
      }
    } catch (error) {
      console.error(error);
      dispatch({ type: actions.profile.DATA_FETCHING, data: error.message });
    }
  };
  return (
    <div className="relative mb-8 max-h-[180px] max-w-[180px] rounded-full lg:mb-11 lg:max-h-[218px] lg:max-w-[218px]">
      <img
        className="w-full h-full object-cover rounded-full aspect-square"
        src={`${import.meta.env.VITE_SERVER_BASE_URL}/${state?.user?.avatar}`}
        alt={state?.user?.firstName}
      />

      <form>
        <button
          onClick={handleImageUpload}
          className="flex-center absolute bottom-4 right-4 h-7 w-7 rounded-full bg-black/50 hover:bg-black/80 cursor-pointer"
          type="submit"
        >
          <img src={EditIcon} alt="Edit" />
        </button>

        <input id="file" type="file" hidden ref={fileUploadRef} />
      </form>
    </div>
  );
};
